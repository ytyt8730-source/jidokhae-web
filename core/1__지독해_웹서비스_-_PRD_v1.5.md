## 1. 배경 및 맥락

### 현재 상황

지독해는 경주/포항에서 운영되는 독서모임이다. 총 회원 250명 중 활성 회원 약 80명이 월 8회(경주 4회, 포항 4회) 정기모임에 참여하고 있다. 현재 운영은 카카오톡 오픈채팅, 구글폼, 구글스프레드시트, 계좌이체를 조합하여 진행 중이다.

### 문제점

**회원 측면:**
- 모임 신청 과정이 번거롭다 (입금자명 변경 → 계좌이체 → 댓글 신청)
- 취소/환불 요청 시 운영자 개인 연락 후 처리까지 시간이 오래 걸린다
- 카카오톡 일정이 한눈에 들어오지 않아 모임 존재를 잊기 쉽다
- 신청해놓고 뭘 신청했는지 헷갈린다

**운영자 측면:**
- 월 5~7건의 환불 요청마다 입금 내역을 일일이 확인해야 한다
- 정보가 여러 도구에 흩어져 있어 현황 파악이 어렵다
- 이번 달 수입, 정산 내역을 한눈에 파악하기 어렵다

**성장 측면:**
- 신규 회원의 첫 모임 후 재참여율이 약 50%로, 절반이 이탈하고 있다

### 이 제품이 필요한 이유

위 문제들로 인해 회원들은 모임 참여에 부담을 느끼고, 운영자는 정산과 관리에 과도한 시간을 쓰고 있다. 이는 회원들이 지독해에 **소속감**과 **자부심**을 느끼기 어렵게 만드는 핵심 요인이다.

## 2. 목적 및 목표

### 본질

회원들의 부담을 줄여주고, 지독해에 **심미적 소속감과 자부심**을 느끼게 해주는 것

> "바쁜 현생을 사는 회원들이 **Frictionless** 경험으로 자연스럽게 스며들고,
> 낮과 밤의 서재를 오가며 지적인 충만함을 느끼게 한다."

### 핵심 가치

| 가치 | 키워드 | 설명 |
|------|--------|------|
| **Frictionless** | 쉬운 연결 | 페이지 이동 없는 Bottom Sheet와 Sticky CTA로 물 흐르듯 신청 |
| **Atmosphere** | 공간감 | Electric & Warm 모드를 통해 사용자의 기분에 맞춘 서재 경험 |
| **Pride** | 소속감 | `<Trophy />`와 Glow 효과로 연출되는 고급스러운 성취 경험 |
| **Trust** | 신뢰 | `<KongIcon />` 기반의 투명하고 직관적인 화폐 시스템 |

### 측정 가능한 목표

**핵심 지표**

| 항목 | 현재 | 목표 |
|------|------|------|
| 신규 회원 재참여율* | 약 50% | 70% 이상 |
| 운영자 환불 처리 개입 | 월 5~7건 수동 처리 | 0건 (셀프 처리) |
| 모임 신청 단계 | 4단계 | 2단계 (3-Click Rule) |
| 정산 현황 확인 | 여러 도구 확인 필요 | 대시보드 1곳에서 확인 |

*신규 회원 재참여율: 첫 정기모임 참여 완료 후 60일 내 두 번째 모임 참여 완료한 회원 비율

**추가 지표**

| 항목 | 측정 방법 |
|------|-----------|
| 리마인드 후 노쇼율 | 리마인드 발송 후 미참여 건수 / 전체 신청 건수 |
| 월말 리마인드 후 신청 전환율 | 월말 리마인드 발송 후 7일 내 신청한 회원 수 / 리마인드 대상 회원 수 |
| 칭찬하기 참여율 | 칭찬 제출 건수 / 참여 완료 건수 |
| 온보딩 이탈률 | 첫 참여 후 60일 이상 미참여 회원 / 첫 참여 완료 전체 회원 |
| 휴면 복귀율 | 휴면 위험 리마인드 후 30일 내 참여 완료한 회원 / 리마인드 대상 회원 |

## 3. 주요 사용자 스토리

### 기존 회원

- "기존 회원으로서, 다가오는 모임 일정을 한눈에 확인하고 싶다. 그래서 참여할 모임을 쉽게 결정할 수 있다."
- "기존 회원으로서, **페이지 이동 없이 한 화면에서** 모임 정보를 확인하고 결제하고 싶다. (3-Click Rule)"
- "기존 회원으로서, 직접 취소하고 환불받고 싶다. 그래서 운영자에게 연락하고 기다리는 번거로움을 피할 수 있다."
- "기존 회원으로서, 모임 전에 리마인드를 받고 싶다. 그래서 신청해놓고 잊어버리는 일이 없다."
- "기존 회원으로서, 내 참여 기록과 배지를 보고 싶다. 그래서 지독해와 함께한 시간을 느낄 수 있다."
- "기존 회원으로서, 모임에서 좋았던 사람에게 감사를 표현하고 싶다. 그래서 따뜻한 마음을 전할 수 있다."
- "기존 회원으로서, 함께 읽은 책들을 기록하고 싶다. 그래서 독서 여정을 쌓아갈 수 있다."
- "기존 회원으로서, 함께 읽은 책에 대한 짧은 생각을 기록하고 싶다. 그래서 나중에 어떤 생각으로 이 책을 읽었는지 되돌아볼 수 있다."
- "기존 회원으로서, 누가 모임에 신청했는지 궁금하다. 그래서 아는 사람이 있는지 확인할 수 있다."
- "기존 회원으로서, 모임 후기를 남기고 싶다. 그래서 다른 회원들에게 모임의 분위기를 알려줄 수 있다."
- "기존 회원으로서, **내 취향에 따라 앱의 분위기(Warm/Electric)를 바꾸고 싶다.**"

### 신규 회원

- "신규 회원으로서, 지독해가 어떤 곳인지 먼저 알고 싶다. 그래서 나와 맞는 모임인지 판단할 수 있다."
- "신규 회원으로서, 첫 모임 전에 안내를 받고 싶다. 그래서 낯선 곳에 가는 부담을 줄일 수 있다."
- "신규 회원으로서, 첫 모임 후 경험을 나누고 싶다. 그래서 지독해와의 연결을 이어갈 수 있다."

### 운영자

- "운영자로서, 이번 달 참가 현황과 수입을 한눈에 보고 싶다. 그래서 정산에 드는 시간을 줄일 수 있다."
- "운영자로서, 환불이 자동으로 처리되길 원한다. 그래서 반복적인 수동 작업에서 벗어날 수 있다."
- "운영자로서, 회원들에게 알림을 보내고 싶다. 그래서 중요한 공지를 놓치지 않게 전달할 수 있다."
- "운영자로서, 다른 운영진에게 일부 권한만 부여하고 싶다. 그래서 역할을 나눠서 운영할 수 있다."
- "운영자로서, 자동 알림의 문구와 발송 시점을 조정하고 싶다. 그래서 상황에 맞게 유연하게 운영할 수 있다."

## 4. 성공 지표

### 정량적 지표

- 신규 회원 첫 모임 후 재참여율: 50% → 70% (첫 정기모임 참여 완료 후 60일 내 두 번째 모임 참여 완료 기준)
- 운영자 환불 처리 개입 건수: 월 5~7건 → 0건
- 모임 신청 완료까지 소요 단계: 4단계 → 2단계 (3-Click)
- 리마인드 후 노쇼율 감소
- 월말 리마인드 후 신청 전환율
- 칭찬하기 참여율
- 온보딩 이탈률 (첫 참여 후 60일 이상 미참여)
- 휴면 복귀율 (휴면 리마인드 후 30일 내 참여)

### 정성적 지표

- 회원 피드백: "모임 신청이 편해졌다", "잊지 않고 참여할 수 있다"
- 운영자 체감: "정산 스트레스가 줄었다", "환불 처리에 신경 안 써도 된다"
- 브랜드 인식: "지독해가 이전보다 체계적으로 느껴진다" (설문 측정)

### 회원 세그먼트 정의

회원 생애주기에 따른 세그먼트 분류. 리마인드 대상 선정 및 지표 측정에 활용한다.

**신규 회원 정의:** 가입 완료 후 첫 정기모임 참여 완료 전까지. 첫 정기모임 참여 완료 시 기존 회원으로 자동 전환.

| 세그먼트 | 조건 | 리마인드 전략 |
|----------|------|--------------|
| 신규 | 가입 완료, 첫 정기모임 참여 완료 전 | 첫 모임 안내 |
| 온보딩 중 | 첫 정기모임 참여 완료 후 30일 이내 | 재참여 유도 (첫 참여 3일 후) |
| 온보딩 이탈 위험 | 첫 참여 후 30~60일, 두 번째 참여 없음 | 복귀 유도 (45일 경과 시) |
| 온보딩 이탈 | 첫 참여 후 60일 이상, 두 번째 참여 없음 | - |
| 성장 중 | 참여 완료 2~4회, 최근 3개월 내 참여 | 월말 참여 독려 |
| 충성 (활성) | 참여 완료 5회 이상, 최근 3개월 내 참여 | 월말 참여 독려 |
| 휴면 위험 | 마지막 참여 완료 3~5개월 전 (모든 모임 기준) | 복귀 유도 (3개월 경과 시) |
| 자격 만료 임박 | 마지막 정기모임 참여 완료 5~6개월 전 | 긴급 복귀 유도 (5개월 경과 시) |
| 자격 만료 | 마지막 정기모임 참여 완료 6개월 이상 전 | 정기모임 재참여 안내 |

※ "참여"는 모두 "참여 완료" 기준. 신청만 하고 취소/노쇼는 참여로 카운트하지 않음.
※ 세그먼트는 상호 배타적이지 않음. 예: "충성 + 휴면 위험" 동시 해당 가능.
※ "자격 만료 임박"은 정기모임 기준, "휴면 위험"은 모든 모임 기준으로 다름.

## 5. 제품 개념 및 주요 기능

### 디자인 원칙 (Design Principles)

| 원칙 | 설명 | 구현 |
|------|------|------|
| **No-Emoji** | 모든 이모지 사용 금지 | `Lucide React` 아이콘 또는 커스텀 SVG (`<KongIcon />`) 사용 |
| **One-Page** | 상세 페이지 이동 지양 | **Bottom Sheet** 활용, 화면 전환 최소화 |
| **3-Click Rule** | 신청 완료까지 3번의 탭 이내 | Sticky CTA, 단계 축소 |
| **Mood-Switchable** | 두 가지 테마 제공 | Electric Mode / Warm Mode (CSS Variables) |

---

### MVP 핵심 기능 (반드시 포함)

#### 회원 인증
- 카카오 로그인
- 이메일/비밀번호 로그인 및 회원가입

#### 홈 화면 (Dashboard)

**레이아웃: Bento Grid**
- 주요 정보를 그리드 형태로 구조화
- 콘텐츠/홍보 배너
- 이번 달 일정 (컴팩트 카드형)
- 이번 주 모임 하이라이트 표시
- 내 신청 모임 + D-day 표시 (예: "D-3")

**애니메이션: Staggered Reveal**
- 페이지 진입 시 일정 카드가 100ms 간격으로 순차적으로 등장
- Framer Motion `staggerChildren` 활용

**테마 스위처**
- 사이드바 하단(PC) 또는 설정(Mobile)에서 Electric/Warm 모드 전환
- `data-theme` 속성 기반 CSS Variables 전환

#### 일정 확인
- 모임 목록 (캘린더 뷰, 컴팩트 카드 뷰)
- 모임 상태 표시: 모집중, 마감임박(남은 자리 3명 이하), 마감, 대기가능

#### 모임 정보 (Bottom Sheet)

**트리거:** 일정 카드 탭 시 화면 하단에서 올라오는 시트

**배경: Atmospheric Blur**
- 모임 커버 이미지 또는 책 표지를 배경으로 활용
- `backdrop-filter: blur()` 적용하여 깊이감 조성

**정보 표시:**
| 정보 | 아이콘 | 비고 |
|------|--------|------|
| 모임명 | - | 헤드라인 타이포그래피 |
| 일시 | `<Calendar />` | - |
| 장소 | `<MapPin />` | 모임마다 다를 수 있음 |
| 참가 인원 | `<Users />` | 정원은 숨김, "O명 참여"만 표시 |
| 참가비 | `<KongIcon />` | 예: `[KongIcon] 10,000콩` |
| 모임 설명 | - | 안내 문구 |
| 환불 규정 | - | 모임 유형별로 다름 |

**정원 표시 규칙:**
- 기본: "O명 참여"만 표시 (정원은 표시하지 않음)
- 마감 임박 시 (남은 자리 3명 이하): "마감 임박" 뱃지 표시 (Pulse 애니메이션)
- 마감 시: "마감" 표시

#### 참가자 목록
- 기본 화면: 참가 인원 숫자만 표시
- 클릭 시: 참가 인원 3명 이상 시 참가자 일부만 공개
  - 누적 칭찬 10개 이상 받은 참가자들 중 랜덤으로 우선 표시
  - 그 외 참가자는 칭찬 많이 받은 순으로 일부만 표시
  - 나머지 참가자는 흐릿하게 처리 (보이지 않음)

#### 환불 규정 기본값
- 정기모임: 3일 전 100%, 2일 전 50%, 이후 불가
- 토론모임: 2주 전 100%, 7일 전 50%, 이후 불가
- 기타 모임: 생성 시 직접 설정 가능

#### 신청/결제 (Frictionless Flow)

**Sticky CTA**
- Bottom Sheet 하단에 고정된 버튼
- 시트 스크롤과 무관하게 항상 노출
- 버튼 텍스트: `[KongIcon] {금액}콩으로 신청하기`

**결제 방법:**
- **간편결제** (카카오페이, 토스페이먼츠 등): 즉시 결제 → 참가 확정
- **계좌이체**: 계좌 정보 확인 → 입금 완료 체크 → 입금대기 상태 → 운영자 확인 후 참가 확정

**계좌이체 세부 사항:**
- 입금자명 형식: "MMDD_이름" (예: 0125_홍길동)
- 계좌 정보 클립보드 복사 기능
- 입금 완료 체크 시 정원 즉시 예약 (다른 사용자 신청 차단)
- 24시간 미입금 시 자동 취소 및 정원 복구
- 운영자 입금 확인 후 참가 확정 전환

#### 대기 신청
- 정원 초과 시 대기 등록 (결제 없이 대기만)
- 자리 발생 시 대기자에게 알림
- 알림 받은 대기자가 직접 결제하여 확정
- 미응답 시 다음 대기자에게 순차 알림
- 응답 대기 시간:
  - 모임 3일 전 이전: 24시간
  - 모임 3일 전 ~ 1일 전: 6시간
  - 모임 1일 전 이후: 2시간

#### 취소/환불
- 셀프 취소
- 환불 처리 방식:
  - **간편결제**: PG사 API를 통한 자동 환불 (운영자 개입 없음)
  - **계좌이체**: 환불 계좌 수집 (은행, 계좌번호, 예금주) → 운영자 수동 환불 후 완료 처리
- 해당 모임의 환불 규정에 따라 자동 계산
- 취소 사유 입력 필수 (운영 개선용 데이터 수집 목적)

#### 리마인드 (자동 발송)

**발송 시점:**
- 모임 리마인드: 3일 전, 1일 전, 당일
- 월말 정기 리마인드: 해당 월에 신청 모임이 없는 회원에게 이름 포함한 참여 독려 알림
- 리마인드 시간은 오전 7~8시

**No-Emoji 알림톡 가이드라인:**

| 상황 | Before (금지) | After (권장) |
|------|---------------|--------------|
| 모임 임박 | "내일 모임이에요! 잊지 마세요" | "모임이 하루 앞으로 다가왔습니다." |
| 칭찬 도착 | "누군가 칭찬을 보냈어요!" | "따뜻한 마음이 전해졌습니다." |
| 배지 획득 | "축하해요! 배지를 획득했어요" | "새로운 배지가 서재에 추가되었습니다." |
| 환불 완료 | "환불 완료! 다음에 또 만나요" | "환불이 완료되었습니다. 다음 모임에서 뵙겠습니다." |

#### 참여 완료 판단
- 모임 종료 3일 후 알림 발송: "[모임명] 모임 어떠셨어요?"
- 선택지: [칭찬하기] [참여했어요] [후기 남기기]
- 셋 중 하나라도 선택하면 → 참여 완료 처리
- 7일 지나도 아무 액션 없으면 → 자동 참여 완료 처리
- 노쇼(결제했지만 미참석)는 운영자가 "미참여"로 수동 변경

#### 마이페이지 (My Growth)

**Electric Mode 스타일:**
- 힙한 그라디언트 배경
- Sans-serif 숫자 강조
- Accent Color (Lime) Glow 효과

**Warm Mode 스타일:**
- 차분한 종이 질감 배경 (Noise 텍스처)
- Serif 문구
- Accent Color (Orange) Subtle Glow

**표시 정보:**
- 내 신청 모임 목록
- 참여 기록: 총 참여 횟수, 연속 참여, "함께한 지 OOO일째"
- 정기모임 자격 상태 표시: "마지막 참여: O월 O일"
- 6개월 임박 시 자격 만료 예정 리마인드

#### 배지 시스템 (Pride System)
- 첫 참여, 10회, 연속 4주 등 달성 시 배지 부여
- **획득 연출:**
  - `<Trophy />` Lucide 아이콘 사용
  - **Confetti Canvas 효과** (react-confetti)
  - **3D 카드 회전** (Framer Motion `rotateY`)
  - 테마별 Accent Color Glow
- 배지는 본인만 볼 수 있음

#### 칭찬하기 (Pride System)
- 모임 종료 3일 후 알림으로 칭찬하기 유도
- 익명으로 진행
- **UI: 텍스트 중심의 우아한 디자인** (이모지/하트 아이콘 사용 금지)
- 선택형 문구:
  - 덕분에 좋은 시간이었어요
  - 이야기가 인상 깊었어요
  - 따뜻한 분위기를 만들어주셨어요
  - 새로운 관점을 얻었어요
  - 다음에도 함께하고 싶어요
- **Interaction:** 칭찬 전송 시 부드러운 빛 퍼짐 효과 (Framer Motion)
- 한 모임당 1명만 칭찬 가능
- 같은 사람에게는 3개월 내 중복 칭찬 불가
- 칭찬이 쌓이면 배지 부여

#### 건의하기
- 모임 종료 후 건의/피드백 제출 기능
- 칭찬하기와 함께 제공

#### 후기 남기기
- 모임 종료 후 모든 회원이 후기 작성 가능 (신규/기존 무관)
- 신규 회원 첫 모임 후에는 후기 작성을 더 강하게 유도
- 후기 작성 시 "후킹페이지에 익명으로 공개" 옵션 제공
- 공개 동의한 후기만 후킹 랜딩페이지에 표시

#### 내 책장 (MVP)
- 함께 읽은 책 등록 (제목, 저자) - `<BookOpen />` 아이콘 활용
- 한 문장 기록 (선택): "이 책에서 기억에 남는 것은..."
- 내 책장 목록 확인 (한 문장 기록과 함께 표시)
- 총 등록 책 수 표시
- 같은 책 중복 등록 방지
- 참여 완료 후 책장 추가 시 한 문장 기록 유도

#### 신규 회원 플로우

신규 회원은 첫 정기모임 참여 완료 전까지로 정의한다. (상세 정의는 섹션 4 "회원 세그먼트 정의" 참조)

- 후킹 랜딩페이지: 브랜드 스토리, 분위기, 회원 후기 (Bento Grid 레이아웃)
- 신규 회원이 일정 클릭 시 안내 Bottom Sheet 표시: "지독해가 어떤 곳인지 먼저 보시는 걸 추천해요."
- 선택지 제공: [브랜드 스토리 보러가기] / [바로 신청하기]
- 첫 모임 전날 환영 알림: 준비물, 장소 안내
- 첫 모임 참여 완료 후 알림: "첫 모임 어떠셨어요? 후기를 남겨주세요!" + 다음 정기모임 안내

#### 정기모임 자격 규정
- 모든 모임(토론모임 등) 참여의 전제조건: 6개월 이내 정기모임 참여 기록
- 자격 미충족 회원이 다른 모임 신청 시: 정기모임 먼저 참여하라는 안내와 함께 정기모임 목록으로 유도

#### 운영자 기능
- 대시보드: 이번 달 참가 현황, 수입, 환불 내역, 재참여율, 입금대기 건수, 환불대기 건수
- 모임 생성/수정/삭제 (환불 규정 포함)
- **입금 확인 관리**:
  - 입금대기 목록 조회 (모임별/전체)
  - 입금자명으로 검색하여 매칭
  - 입금 확인 → 참가 확정 처리
  - 입금자명 불일치 시 수동 매칭 기능
- **환불 관리**:
  - 환불대기 목록 조회
  - 환불 계좌 정보 확인
  - 환불 완료 처리
- 요청함: 문의 확인 및 답변 기능
- 알림톡 수동 발송: 공지사항, 이벤트 안내 등 운영자가 원할 때 발송
- 홈 화면 콘텐츠 배너 관리
- 노쇼 회원 "미참여" 처리

#### 알림 템플릿 관리 (운영자)
- 자동 발송 알림의 문구를 운영자가 수정 가능
- 알림 종류별로 템플릿 관리:
  - 모임 리마인드 (3일 전, 1일 전, 당일)
  - 월말 참여 독려
  - 대기자 자리 발생 알림
  - 첫 모임 환영 알림
  - 모임 후 "어떠셨어요?" 알림
  - 정기모임 자격 만료 임박 알림
  - 온보딩 이탈 위험 알림
  - 휴면 위험 알림
- 발송 시점 조정 가능 (예: 3일 전 → 2일 전으로 변경)
- 특정 알림 on/off 가능

#### 운영자 권한 관리
- 메인 운영자: 모든 권한 보유
- 다른 운영진에게 특정 권한만 선택적으로 부여 가능

  - 모임 생성/수정/삭제
  - 입금/환불 관리
  - 알림톡 수동 발송
  - 배너 관리
  - 요청함 답변
  - 대시보드 열람
  - 알림 템플릿 관리

### 핵심 사용자 흐름 (One-Page Flow)

**기존 회원 (3-Click Flow):**
1. 웹 접속 (Electric/Warm 모드 자동 또는 수동 선택)
2. 홈: Bento Grid 대시보드 + Staggered Animation으로 일정 카드 순차 등장
3. **[TAP 1]** 일정 카드 클릭 → Bottom Sheet 오픈 (페이지 이동 없음)
4. (선택) 참가 인원 클릭 → 참가자 일부 확인
5. **[TAP 2]** 하단 Sticky CTA "[KongIcon] 10,000콩으로 신청하기"
6. **[TAP 3]** 포트원 결제 창에서 결제 완료
7. Bottom Sheet 닫히며 "참여 확정" Toast 메시지
8. 리마인드 알림 (모임 3일 전, 1일 전, 당일)
9. 모임 참여
10. 모임 3일 후 → "[모임명] 모임 어떠셨어요?" 알림 → [칭찬하기] [참여했어요] [후기 남기기] 선택

**신규 회원:**
1. 웹 접속 → 일정 화면
2. 일정 카드 탭 → 안내 Bottom Sheet 오픈: [브랜드 스토리 보러가기] / [바로 신청하기]
3. 브랜드 스토리 선택 시: Bento Grid 랜딩페이지 확인 → 신청
4. 바로 신청 선택 시: Bottom Sheet 내 결제 진행
5. 첫 모임 전날 환영 알림 (준비물, 장소 안내)
6. 첫 모임 참여
7. 첫 모임 3일 후 → "첫 모임 어떠셨어요? 후기를 남겨주세요!" + 다음 정기모임 안내

**취소/환불:**
1. 마이페이지 → 신청 내역
2. 취소하기 클릭
3. 마음 돌리기 Bottom Sheet 오픈 (함께 신청한 회원, 모임 임박 등 표시)
4. 취소 사유 입력 (필수)
5. 해당 모임 환불 규정에 따라 자동 환불 처리
6. 대기자 있으면 자동으로 다음 대기자에게 알림

### 아이콘 시스템 (No-Emoji 정책)

모든 UI에서 이모지 사용을 금지하고, Lucide React 아이콘을 사용한다.

| 용도 | 아이콘 컴포넌트 | import |
|------|----------------|--------|
| 참가비 (콩) | `<KongIcon />` | 커스텀 SVG |
| 배지/트로피 | `<Trophy />` | `lucide-react` |
| 일정/캘린더 | `<Calendar />` | `lucide-react` |
| 장소/위치 | `<MapPin />` | `lucide-react` |
| 책/책장 | `<BookOpen />` | `lucide-react` |
| 알림 | `<Bell />` | `lucide-react` |
| 설정 | `<Settings />` | `lucide-react` |
| 사용자/참가 인원 | `<Users />` | `lucide-react` |
| 체크/완료 | `<Check />` | `lucide-react` |

**Lucide 설정:**
- `strokeWidth={1.5}`
- `size={16~24}`

### 애니메이션 사양 (Motion)

| 상황 | 효과 | 구현 방식 |
|------|------|---------|
| 페이지 진입 시 카드 리스트 | 아래→위 Stagger (100ms 간격) | Framer Motion `staggerChildren` |
| Bottom Sheet 오픈 | Spring 물리 효과 (쫀득한 느낌) | Framer Motion `type: "spring"` |
| 카드 호버 | Y축 -4px 이동, 그림자 확대 | `whileHover={{ y: -4 }}` |
| 카드 탭 | scale 0.98 (눌림 느낌) | `whileTap={{ scale: 0.98 }}` |
| 마감 임박 뱃지 | 부드러운 Pulse (1.5초 주기) | CSS animation 또는 Framer Motion |
| 배지 획득 | Confetti Canvas + 3D 카드 회전 | `react-confetti` + `rotateY` |
| 칭찬 완료 | 빛 퍼짐 효과 | Framer Motion opacity + scale |
| 연속 참여 표시 | 프로그레스 바 채워짐 | width 0% → 현재값 애니메이션 |

### 마이크로 인터랙션 원칙
- 모든 클릭 가능 요소는 호버/탭 피드백 필수
- 전환 시간: 150~300ms (자연스러운 속도)
- 이징 함수: `ease-out` 기본, 바운스 효과는 게이미피케이션에만 사용
- 피드백은 즉각적이되, 과하지 않게

### 그리드 시스템
- **8px 그리드:** 모든 여백, 크기는 8의 배수 (8, 16, 24, 32, 48, 64...)
- 컴포넌트 간 간격: 16px (작음), 24px (보통), 32px (큼)
- 페이지 패딩: 모바일 16px, 데스크톱 24px

### 레퍼런스 브랜드
- **Apple:** 깔끔한 여백, 타이포그래피 강조, 미니멀한 인터랙션
- **Airbnb:** 따뜻한 톤, 사용자 중심 마이크로인터랙션, 친근한 UX

## 7. 시스템 요구사항

### 플랫폼

- 1차: 웹서비스 (모바일 최적화 반응형, One-Page App 구조)
- 추후: 앱 전환 고려 (웹뷰 또는 네이티브)

### 외부 연동

- 결제: 간편결제 PG 연동 (부분 환불 API 포함)
- 알림: 카카오 알림톡 연동 (카카오톡 채널 보유, 사업자등록 완료)
- 인증: 카카오 로그인 API

### 데이터 요구사항

**회원 정보:**
- 이름, 이메일, 전화번호, 가입일, 인증 방식(카카오/이메일), 신규 회원 여부, 첫 정기모임 참여일, **선호 테마 (Electric/Warm)**

**모임 정보:**
- 모임명, 모임 유형, 일시, 장소, 정원, 참가비(콩), 모임 설명/안내 문구, 환불 규정, 상태, **커버 이미지 URL**

**신청 정보:**
- 회원-모임 매핑, 결제 방법(간편결제/계좌이체), 결제 상태, 결제 금액, 취소 여부, 취소 사유, 취소 일시, 환불 금액
- 계좌이체 전용: 입금자명, 입금 기한, 환불 계좌 정보(은행/계좌/예금주), 환불 완료 일시

**대기 정보:**
- 대기 순번, 알림 발송 여부, 알림 발송 일시, 응답 기한

**참여 기록:**
- 참여 완료 여부, 완료 처리 방식(칭찬/건의/후기/참여확인/자동/운영자), 누적 참여 횟수, 연속 참여 기록

**배지:**
- 배지 종류, 획득 일시

**칭찬:**
- 칭찬 발신자, 수신자, 모임, 선택 문구, 일시 (발신자-수신자 간 3개월 제한 체크용), 누적 칭찬 수

**책장:**
- 회원, 책 제목, 저자, 등록일, 한 문장 기록 (선택)

**후기:**
- 회원, 모임, 후기 내용, 작성일, 공개 동의 여부

**알림 템플릿:**
- 알림 종류, 문구, 발송 시점, 활성화 여부

**알림 발송 기록:**
- 알림 종류, 수신자(회원), 발송 일시, 발송 결과(성공/실패), 관련 모임(있는 경우)

**운영자 권한:**
- 운영자 계정, 권한 목록 (모임 관리, 입금/환불 관리, 알림 발송, 배너 관리, 요청함 답변, 대시보드 열람, 알림 템플릿 관리)

### 권한 구조

- 일반 회원: 일정 확인, 신청/취소, 마이페이지, 칭찬/건의/후기, 내 책장, 참가자 목록 확인, **테마 변경**
- 운영자: 부여받은 권한에 따라 기능 접근
- 메인 운영자: 모든 권한 + 다른 운영자 권한 관리 + 노쇼 회원 미참여 처리 + 알림 템플릿 관리

## 8. 가정, 제약 사항 및 의존성

### 가정 (Assumptions)

- 대부분의 회원은 모바일 환경에서 서비스를 이용한다
- 회원들은 카카오톡을 사용하고 있어 알림톡 수신이 가능하다
- 기존 카카오톡 오픈채팅방은 당분간 병행 운영될 수 있다
- 기존 회원도 새로 가입하여 서비스를 시작한다 (데이터 마이그레이션 없음)
- 노쇼는 드물게 발생하며, 운영자가 파악 가능하다

### 제약 (Constraints)

**디자인 제약:**
- **이모지 사용 절대 금지 (Strict No-Emoji Policy)**
- 모든 아이콘은 `Lucide React` 라이브러리 사용을 원칙으로 함
- 콩 아이콘은 지정된 `KongIcon.tsx` (SVG) 컴포넌트만 사용
- 테마 전환은 `darkMode` 클래스가 아닌 CSS Variables (`data-theme`)로 구현
- Electric Mode에서 Lime 색상은 배경으로만 사용, 텍스트 금지

**비즈니스 제약:**
- 환불 규정은 모임 유형별로 다름 (정기모임: 3일/2일, 토론모임: 2주/7일)
- 정기모임 참여 자격 규정: 6개월 이내 정기모임 참여 기록 필요
- 모임 정원: 기본 14명
- 칭찬 규칙: 한 모임당 1명, 같은 사람에게 3개월 내 중복 불가, 익명, 선택형
- 참가자 목록 공개: 누적 칭찬 10개 이상 회원 우선 표시, 전체 공개 안 함
- 대기자 응답 대기 시간: 모임 임박도에 따라 24시간/6시간/2시간
- 참여 완료 미응답 시 7일 후 자동 처리
- 결제 정보(카드 번호 등)는 서비스에서 직접 저장하지 않음 (PG사에 위임)
- 참가비 표시 단위: 콩 (반드시 `<KongIcon />`과 함께)
- 참가 인원 표시 시 정원은 표시하지 않음

### 의존성 (Dependencies)

- PG사 API 연동 및 승인 (결제, 부분 환불 기능 포함)
- 카카오 개발자 계정 및 로그인 API
- 카카오톡 채널 및 알림톡 발송 승인

## 9. 릴리즈 계획 및 실험 계획

### 릴리즈 단계

| 단계 | 대상 | 목적 |
|------|------|------|
| Alpha | 운영자 + 친한 회원 3~5명 | 핵심 흐름 검증, 버그 발견 |
| Beta | 활성 회원 20~30명 | 실제 모임에 적용, 피드백 수집 |
| 정식 출시 | 전체 회원 | 기존 프로세스 대체 |

### 검증 계획

- Beta 기간 중 "신청 완료까지 걸린 시간/단계" 측정
- Beta 사용자 대상 간단 설문: "기존 방식 대비 편의성" (1~5점)
- Beta/정식 출시 후 회원 설문: "지독해가 이전보다 체계적으로 느껴지나요?" (1~5점)
- 정식 출시 후 1개월: 신규 회원 재참여율 측정
- 정식 출시 후 1개월: 월 환불 건 중 운영자 개입 건수 측정
- 정식 출시 후 1개월: 리마인드 후 노쇼율 측정
- 정식 출시 후 1개월: 월말 리마인드 후 신청 전환율 측정
- 정식 출시 후 1개월: 칭찬하기 참여율 측정
- 정식 출시 후 3개월: 온보딩 이탈률, 휴면 복귀율 측정
