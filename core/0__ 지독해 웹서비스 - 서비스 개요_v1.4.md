## 1. 서비스명
**지독해 (가칭: 지독해 웹)**

---

## 2. 서비스 한 줄 정의
> 지독해 회원들이 모임과 우아하게 연결되어 있다는 감각을 유지할 수 있게 해주는, **두 가지 무드(Electric & Warm)의 독서 리추얼 플랫폼**

---

## 3. 해결하려는 문제

**회원 관점**
- 카카오톡 일정이 한눈에 안 들어와서 모임 확인이 불편함
- 참가 신청(입금자명 수정 → 계좌이체 → 댓글)이 번거로움
- 취소/환불 요청 시 개인 연락 → 확인 → 처리까지 시간이 오래 걸림
- 바쁜 일상에서 신청해놓고 까먹거나, 뭘 신청했는지 헷갈림
- 리마인드가 없으면 모임 존재 자체를 잊어버림

**운영자 관점**
- 환불 요청 시 입금 내역을 일일이 찾아서 확인해야 함
- 이번 달 수입, 공간대여비, 재참여율 등 현황 파악이 어려움
- 정보가 카톡, 구글시트, 은행앱 등에 흩어져 있어 기억에 의존해야 함

---

## 4. 본질

**표면:** 복잡한 프로세스를 간소화하고 싶다.

**본질:** 회원들에게 '지독해'라는 브랜드에 대한 **심미적 소속감**과 **자부심**을 느끼게 해주고 싶다.

> "바쁜 현생을 사는 회원들이 **Frictionless(마찰 없는)** 경험으로 자연스럽게 스며들고, 낮과 밤의 서재를 오가며 지적인 충만함을 느끼게 한다."

---

## 5. 핵심 가치

| 가치 | 키워드 | 설명 |
|------|--------|------|
| **쉬운 연결** | Frictionless | 페이지 이동 없는 **Bottom Sheet**와 **Sticky CTA**로 물 흐르듯 신청 |
| **분위기** | Atmosphere | **Electric & Warm** 모드를 통해 사용자의 기분에 맞춘 공간감 제공 |
| **소속감** | Pride | `<Trophy />`와 Glow 효과로 연출되는 고급스러운 성취 경험 |
| **신뢰** | Trust | `<KongIcon />` 기반의 투명하고 직관적인 화폐 시스템 |

## 6. 타겟 사용자

**주 사용자: 지독해 기존 회원**
- 정기모임에 참여하거나 참여하려는 회원
- 바쁜 일상 속에서 독서모임을 유지하고 싶은 사람

**부 사용자: 신규 회원**
- 지독해를 처음 알게 된 사람
- 후킹 랜딩페이지를 통해 유입되어 첫 정기모임을 신청하는 사람

**운영자: 영탁님(단무지)**
- 모임 운영, 정산, 회원 관리를 담당
- 다른 운영진에게 일부 권한 위임 가능

---

## 7. 핵심 규칙 (서비스에 반영)

**No-Emoji 정책**
- 서비스 내 모든 이모지 사용을 금지하며, Lucide 아이콘과 커스텀 SVG로 대체한다.

**참여 자격**
- 모든 모임 참여의 전제조건 = 정기모임 참여 이력
- 6개월 이내 정기모임 참여 기록 없으면 다른 모임 참여 불가

**환불 규정**
- 정기모임: 3일 전 100% / 2일 전 50% / 이후 불가
- 토론모임: 2주 전 100% / 7일 전 50% / 이후 불가

**화폐 시스템**
- 참가비 단위: **"콩"**
- 반드시 `<KongIcon />`과 함께 표기 (예: `[KongIcon] 10,000콩`)

## 8. 주요 기능

### MVP (1차 개발)

**일정 경험 (Visual Experience)**
- **Staggered Reveal:** 페이지 진입 시 일정 카드가 100ms 간격으로 순차적으로 떠오르는 애니메이션
- **One-Page Flow:** 일정 클릭 시 페이지 이동 없이 **Bottom Sheet**가 부드럽게 올라옴
- **Atmospheric Card:** 모임 커버 이미지를 배경 블러 처리하여 깊이감 조성
- 정보 표기: `<Calendar />` 일시, `<MapPin />` 장소, `<Users />` 참가 인원 (정원은 숨김)

**신청/결제 (Frictionless)**
- **Sticky CTA:** Bottom Sheet 하단에 고정된 버튼으로 즉시 결제
- 버튼 텍스트: `[KongIcon] 10,000콩으로 신청하기`
- 간편결제 (카카오페이, 토스페이먼츠)
- 대기 신청 (자리 나면 알림 → 그때 결제)

**취소/환불 (Self-Service)**
- 셀프 취소 + 자동 환불 (규정에 따라)
- 취소 사유 입력 필수 (운영 개선용)
- 대기자 자동 승격 (취소 발생 → 순차 알림)

**리마인드 (Connection)**
- 자동 리마인드 알림톡 (모임 3일 전, 1일 전, 당일)
- 매월 말 정기 리마인드 (신청 모임 없는 회원에게도)
- 알림 템플릿 관리 (운영자가 문구/발송 시점 수정 가능)
- **No-Emoji:** 알림톡 메시지 내 이모지 사용 금지, 텍스트 위주로 정중하게 작성

**소속감/자부심 (Pride System)**
- **활동 기록:** `<Trophy />` 아이콘과 Accent Color Glow 효과를 활용한 마이페이지
- **배지 시스템:** 배지 획득 시 **Confetti Canvas 효과**와 **3D 카드 회전** 연출 (폭죽 이모지 대신)
- **칭찬하기:** 이모지 없이 타이포그래피와 미세한 빛 퍼짐 모션으로 감사를 전하는 UI
- **내 책장:** `<BookOpen />` 아이콘과 함께 쌓이는 독서 기록, 한 문장 메모

**신규 회원 (Onboarding)**
- 후킹 랜딩페이지 (브랜드 스토리, 분위기, 회원 후기)
- 일정 클릭 시 안내 Bottom Sheet: "지독해가 어떤 곳인지 먼저 보시는 걸 추천해요."
- 첫 모임 전날 환영 알림 (준비물, 장소 안내)
- 첫 모임 후 후기 요청 + 다음 모임 안내

**운영자 기능 (Admin)**
- 대시보드 (이번 달 참가 현황, 수입, 환불 내역, 재참여율)
- 요청함 (환불 요청, 문의 확인 + 답변 기능)
- 홈 화면 콘텐츠 배너 관리
- 알림톡 수동 발송 (공지, 이벤트 안내 등)
- 운영자 권한 관리 (다른 운영진에게 권한 선택적 부여)

---

### 추후 기능 (2차 이후)

- 결제 수단 미리 등록 (원클릭 결제)
- 대기 신청 시 선결제 → 자동 환불
- 구글/애플 캘린더 자동 추가
- 정기모임 개설 요청 (회원들이 원하는 지역/요일 투표)
- 다른 회원 책장 구경
- 책 등록 배지 (10권, 50권 등)
- 참여 리포트 (월말/연말 "올해 OO회 참여")
- 클럽장 콘텐츠 정산 자동화 (80/20 자동 계산)
- 토론모임, 지글챌, 번개모임 전용 기능

## 9. 사용자 경험 흐름 (UX Flow)

**기존 회원 (3-Click Flow)**웹 접속 (Electric/Warm 모드 선택 가능)
→ 홈: Bento Grid 대시보드 + Staggered Animation으로 일정 카드 등장
→ [TAP] 일정 카드 클릭
→ [ACTION] Bottom Sheet 오픈 (페이지 이동 없음)
- Atmospheric Blur 배경
- 모임 상세 정보 표시
→ [TAP] 하단 Sticky CTA "[KongIcon] 10,000콩으로 신청하기"
→ [ACTION] 포트원 결제 창 오버레이
→ 결제 완료 → Bottom Sheet 닫히며 "참여 확정" Toast 메시지
→ (모임 3일 전, 1일 전, 당일) 리마인드 알림톡
→ 모임 참여
→ (3일 후) 칭찬하기/참여했어요/후기 남기기 알림

**신규 회원**웹 접속 → 일정 화면
→ [TAP] 일정 카드 클릭
→ [ACTION] Bottom Sheet 내 안내 모달 오버레이:
"지독해가 어떤 곳인지 먼저 보시는 걸 추천해요."[Link] 브랜드 스토리 보러가기
[Button] 바로 신청하기→ (브랜드 스토리 선택 시) Bento Grid 후킹페이지 확인 → 신청
→ (바로 신청 선택 시) Bottom Sheet 내 결제 진행
→ 첫 모임 전날 환영 알림 (준비물, 장소 안내)
→ 첫 모임 참여
→ (3일 후) "첫 모임 어떠셨어요? 후기를 남겨주세요!" + 다음 정기모임 안내

**취소/환불**마이페이지 → 신청 내역 → 취소하기
→ [ACTION] 마음 돌리기 Bottom Sheet 오픈 (함께 신청한 회원, 모임 임박 등 표시)
→ [TAP] 취소 사유 선택 (필수)
→ [ACTION] 환불 규정 안내 + 동의
→ 최종 확인 후 자동 환불
→ 대기자 있으면 자동 승격 알림

※ 취소는 신청보다 마찰을 높여 충동적 취소 방지 (상세: PRD 섹션 6)

## 10. 기대 효과

**회원**
- 모임 확인/신청/취소가 쉬워져 참여 허들이 낮아짐
- 리마인드로 모임을 잊지 않고, 지독해와 연결된 느낌 유지
- 활동 기록과 책장을 통해 소속감과 자부심
- Electric/Warm 모드로 자신의 기분에 맞는 분위기 선택

**운영자**
- 정산/환불 처리 시간 대폭 단축
- 모든 정보가 한 곳에 → 기억에 의존하지 않아도 됨
- 재참여율 등 데이터 기반 운영 가능
- 알림 문구/시점을 유연하게 조정 가능

**지독해 전체**
- 회원 이탈 감소, 재참여율 증가
- 신규 회원 유입 경로 확보 (후킹 랜딩페이지)
- 브랜드 이미지 강화 ("체계적이고 신뢰할 수 있는 모임")
- "뻔한 AI 디자인"을 탈피한 독창적 브랜드 경험

## 11. 브랜드 톤 (Brand Voice)

**"낮과 밤의 서재 (Day & Night Library)"**

사용자가 선택하는 두 가지 페르소나를 제공하여 몰입감을 극대화한다.

| 모드 | 컨셉 | 컬러 팔레트 | 타이포그래피 | 분위기 |
|------|------|-------------|--------------|--------|
| **Electric Mode** | 힙하고 에너지 넘치는 독서 라운지 | Cobalt Blue (#0047AB) + Acid Lime (#CCFF00) | 고딕체(Sans) 중심: Outfit, Noto Sans KR | 성장과 자극을 주는 공간 |
| **Warm Mode** | 차분하고 지적인 클래식 서재 | Warm Sand (#E8DCC4) + Deep Navy (#1A237E) | 명조체(Serif) 헤드라인: Noto Serif KR | 사유와 위로를 주는 공간 |

**디자인 원칙**
- **No-Emoji:** 모든 이모지 사용 금지. Lucide React 아이콘으로 대체.
- **One-Page:** 페이지 이동 최소화. Bottom Sheet 패턴 활용.
- **3-Click Rule:** 신청 완료까지 3번의 탭 이내.
