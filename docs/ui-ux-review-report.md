## 2. 섹션별 상세 검토

### 2.1 01-landing (홈/소개)

**페이지:** `/`, `/about`

#### 잘 된 점
- 헤드라인 "깊은 사유, 새로운 관점" - 브랜드 메시지 명확
- `heading-themed` 클래스로 테마별 폰트 분기
- Lucide 아이콘 사용 (BookOpen, Users, Heart, MapPin 등)
- Framer Motion 애니메이션 적용
- 반응형 레이아웃

#### 개선 필요
| 이슈 | 심각도 | 위치 | 설명 |
|------|--------|------|------|
| 갤러리 이미지 부재 | Critical | `/about` "이런 분위기예요" | `/images/gallery/` 파일 없음 |
| 모임 카드 썸네일 | Critical | `/` 모임 목록 | 모든 카드 이미지 빈 박스 |
| 후기 폰트 하드코딩 | Warning | `LandingContent.tsx:380` | `font-serif` 고정 |
| CTA 텍스트 불일치 | Warning | `page.tsx` vs `HeroSection.tsx` | "모임 둘러보기" vs "모임 일정 보기" |

### 2.2 02-auth (인증)

**페이지:** `/auth/login`, `/auth/signup`

#### 잘 된 점
- 카카오 로그인 버튼 브랜드 색상(#FEE500) 정확 적용
- 카카오 로고 커스텀 SVG (No-Emoji 준수)
- Lucide 아이콘 사용 (Mail, Lock, User, Phone, AlertCircle)
- 에러/성공 피드백 UI

#### 개선 필요
| 이슈 | 심각도 | 위치 | 설명 |
|------|--------|------|------|
| 플로팅 버튼 | Critical | 좌측 하단 | "멤버십 입장" 버튼 - 디자인 시스템 위반 |
| brand-* 색상 | Warning | 전반 | CSS Variables로 통일 필요 |
| gray-* 색상 | Warning | 전반 | 테마 변수로 통일 필요 |

### 2.4 04-mypage (마이페이지) ⭐ 모범 사례

**페이지:** `/mypage`, `/mypage/tickets`, `/mypage/bookshelf`

#### 잘 된 점 (디자인 시스템 완벽 준수!)
- **테마 토글 위치 (모바일):** 마이페이지 상단 우측 "낮의 서재" + Sun/Moon 토글

- **플로팅 버튼 없음**
- 레벨 배지 "Lv.2 열정멤버" - accent 배경
- 통계 카드 그리드 (총 참여, 연속 참여, 받은 칭찬, 등록한 책)
- 빈 상태 UI (아이콘 + 안내 메시지)

#### 개선 제안 (Enhancement)

|------|------|

### 3.1 플로팅 버튼 수정 가이드

```tsx
// 현재 (잘못됨)
<button className="fixed bottom-4 left-4 ...">
  멤버십 입장
</button>

// 해결: 제거하거나 사이드바로 이동
// 디자인 시스템 v3.4 규칙:
// ❌ 플로팅 버튼 금지 (fixed bottom-4 left-4 등)
// ✅ 사이드바 또는 헤더에 배치
```

### 3.2 콩 아이콘 수정 가이드

```tsx
// 현재 (잘못됨)
<span className="w-2 h-2 rounded-full bg-green-500" />
<span>5,000콩</span>

// 해결
import { KongIcon } from '@/components/icons/KongIcon'

<span className="flex items-center gap-1">
  <KongIcon size={16} />
  <span>5,000콩</span>
</span>
```

```tsx
// 현재 (LandingContent.tsx:380)
<p className="... font-serif">

// 해결
<p className={`... ${theme === 'warm' ? 'font-serif' : 'font-sans'}`}>
```

```tsx
// 현재
className="text-brand-800"
className="text-gray-600"

// 해결
className="text-[var(--text)]"
className="text-[var(--text-muted)]"
```

## 5. 모범 사례

### 5.1 04-mypage 섹션

**04-mypage**는 디자인 시스템 v3.4를 **가장 완벽하게 준수**하고 있습니다.

다른 섹션에서 참고할 점:

3. **레벨 배지 시스템**
   - accent 배경 + 읽기 가능한 텍스트

4. **빈 상태 UI**
   - 아이콘 + 친근한 안내 메시지

## 7. 근본 원인 분석

> 왜 이런 문제들이 반복적으로 발생하는지에 대한 구조적 분석입니다.

### 7.1 식별된 4가지 근본 원인

#### 패턴 1: Admin 예외 마인드셋

**현상:** 관리자 영역에서 디자인 시스템 위반이 더 많이 발견됨

**원인:**
- "관리자는 내부 사용자이므로 UX가 덜 중요하다"는 암묵적 가정
- Admin 페이지 개발 시 빠른 구현을 우선시하여 디자인 시스템 검토 생략
- 예: Admin Dashboard의 `formatCurrency` → `KongIcon` 미적용

**해결책:**
- Admin 페이지도 동일한 디자인 시스템 체크리스트 적용
- `Price` 컴포넌트 등 공유 컴포넌트 사용 의무화

#### 패턴 2: 공유 컴포넌트 부재

**현상:** 동일한 UI 패턴이 여러 파일에서 중복 구현됨

**원인:**
- 브랜드 로고가 `Header.tsx`, `Footer.tsx`, `Ticket.tsx` 등에서 각각 구현
- 테마 전환 로직이 컴포넌트마다 반복됨
- 한 곳 수정 시 다른 곳에서 누락 발생

**해결책:**
- ✅ `BrandLogo` 공유 컴포넌트 생성 완료 (v1.2)
- ✅ `useBrandText()` 훅으로 브랜드명 중앙 관리
- `Price` 컴포넌트를 모든 금액 표시에 의무 사용

#### 패턴 3: 불완전한 레거시 마이그레이션

**현상:** 일부 파일에서 이전 패턴이 그대로 유지됨

**원인:**
- 초기 개발 시 테마 시스템 부재
- 테마 시스템 도입 후 기존 파일 일괄 마이그레이션 미실시
- 예: `font-serif` 하드코딩, `brand-*` 색상 클래스 직접 사용

**해결책:**
- 마이그레이션 스크립트 또는 ESLint 규칙으로 레거시 패턴 감지
- 신규 코드 리뷰 시 테마 변수 사용 여부 체크

#### 패턴 4: 에셋 관리 프로세스 부재

**현상:** 이미지, 갤러리 등 에셋이 placeholder 상태로 방치

**원인:**
- 이미지 에셋 추가를 위한 명확한 프로세스 부재
- "개발 완료 후 나중에 추가" 계획이 실행되지 않음
- 에셋 체크리스트 또는 담당자 미지정

**해결책:**
- 에셋 체크리스트를 Beta 출시 조건에 포함
- `/public/images/` 폴더 구조 표준화
- 에셋 추가 담당자 명시

### 7.2 구조적 개선 완료 현황

| 개선 사항 | 상태 | 설명 |
|-----------|:----:|------|
| BrandLogo 공유 컴포넌트 | ✅ | 테마 인식 로고 컴포넌트 생성 |
| Price 컴포넌트 Admin 적용 | ✅ | Dashboard, Transfers에 KongIcon 적용 |
| LandingContent 폰트 수정 | ✅ | 테마 조건부 font-serif 적용 |
| 갤러리 이미지 폴더 | ✅ | `/public/images/gallery/` 생성 |
| ESLint 규칙 추가 | ⏳ | 향후 권장 |

### 7.3 권장 프로세스 개선

1. **코드 리뷰 체크리스트 추가:**
   - [ ] 브랜드 로고에 `<BrandLogo />` 사용
   - [ ] 금액 표시에 `<Price />` 사용
   - [ ] 테마 의존 스타일에 CSS Variables 사용
   - [ ] 에셋 placeholder 확인

2. **자동화 권장:**
   ```bash
   # ESLint 규칙 예시 (향후 추가)
   - no-hardcoded-brand-text: error
   - require-theme-variables: warn
   - no-formatCurrency-function: error
   ```
