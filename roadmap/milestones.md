# 지독해 웹서비스 MVP - Milestones

---

**문서 버전:** 1.0  
**작성일:** 2026-01-14  
**관련 문서:** 서비스 개요 v1.3 / PRD v1.4 / 기술 스택 v1.2 / 시스템 구조 v1.4

---

## 1. 개요

이 문서는 지독해 웹서비스 MVP 개발을 위한 최상위 전략인 Milestones를 정의합니다.

**개발 방식:** AI 코딩 에이전트 활용 (Cursor, Claude Code 등)  
**예상 총 기간:** 약 12~14주 (3~4개월)  
**목표:** MVP 완성 후 정식 출시

---

## 3. Milestone 상세

> 📋 각 Milestone의 상세 Work Package는 [work-packages/](./work-packages/) 폴더를 참조하세요.

### M1: 프로젝트 기반 구축

**목표:** 개발 인프라 완성, 기본 화면 동작 확인  
**📋 Work Package:** [WP-M1-기반구축.md](./work-packages/WP-M1-기반구축.md)

| 항목 | 내용 |
|------|------|
| **기간** | 1~2주 |
| **선행 조건** | 없음 (시작점) |
| **핵심 가치** | 쉬운 연결 (기반) |

#### 산출물

| 구분 | 산출물 |
|------|--------|
| **인프라** | Next.js 14 프로젝트 초기화 |
| | Supabase 프로젝트 생성 및 연결 |
| | Vercel 배포 파이프라인 구축 |
| | 환경 변수 설정 (dev/prod 분리) |
| **디자인 시스템** | 8px 그리드 시스템 설정 |
| | Pretendard + Noto Serif KR 폰트 로드 |
| | Framer Motion 설치 및 기본 설정 |
| | 공통 애니메이션 variants 정의 |
| **인증** | 이메일/비밀번호 회원가입 |
| | 이메일/비밀번호 로그인 |
| | 로그아웃 |
| **데이터베이스** | 핵심 테이블 생성 (users, meetings, registrations, refund_policies) |
| | Row Level Security 기본 설정 |
| **UI** | 홈 화면 레이아웃 |
| | 모임 카드 컴포넌트 (컴팩트 디자인 + 호버/클릭 애니메이션) |
| | 카드 리스트 Stagger 등장 효과 |
| | 마감 임박 뱃지 Pulse 애니메이션 |
| | 모임 상세 페이지 (정보 표시만) |
| | 마이페이지 기본 (내 정보) |
| | 모바일 반응형 레이아웃 |
| **운영자** | 모임 생성/수정/삭제 기능 |

#### 검증 포인트

- [ ] 회원가입 → 로그인 → 로그아웃 정상 동작
- [ ] 모임 목록이 홈 화면에 컴팩트 카드로 표시됨
- [ ] 홈 화면 진입 시 카드가 순차적으로 나타남 (Stagger)
- [ ] 모임 카드 호버 시 미세하게 떠오름 (Y축 -4px)
- [ ] 마감 임박 뱃지가 부드럽게 깜빡임 (Pulse)
- [ ] Pretendard/Noto Serif KR 폰트가 적용됨
- [ ] 모임 상세 페이지에서 정보 확인 가능
- [ ] 모바일에서 UI가 깨지지 않음
- [ ] Vercel 자동 배포 동작

#### 의존성 다이어그램

```

    ↓
[M1 완료] → M2 시작 가능
```

### M2: 핵심 결제 흐름

**목표:** "신청 → 결제 → 취소 → 환불" 전체 흐름 완성  
**📋 Work Package:** [WP-M2-핵심결제흐름.md](./work-packages/WP-M2-핵심결제흐름.md)

| 항목 | 내용 |
|------|------|
| **기간** | 2~3주 |
| **선행 조건** | M1 완료 |
| **핵심 가치** | 쉬운 연결, 신뢰 |
| **테스트** | Alpha 테스트 (운영자 + 친한 회원 3~5명) |

#### 산출물

| 구분 | 산출물 |
|------|--------|
| **인증 확장** | 카카오 로그인 연동 (Supabase Auth) |
| **결제** | 포트원 연동 (카카오페이/토스페이먼츠) |
| | 모임 신청 + 결제 흐름 |
| | 결제 완료 웹훅 처리 |
| | 결제 상태 실시간 반영 (Supabase Realtime) |
| **환불** | 취소 요청 + 취소 사유 입력 |
| | 환불 규정 자동 계산 (정기모임 3일/2일, 토론모임 2주/7일) |
| | 포트원 환불 API 연동 (부분 환불 포함) |
| | 마음 돌리기 화면 (취소 마찰 증가) |
| **대기** | 정원 초과 시 대기 등록 |
| | 대기자 목록 관리 |
| | 취소 발생 시 수동 알림 (알림톡은 M3) |
| **UI** | 신청 버튼 활성화 |
| | 결제 진행 화면 |
| | 참가 인원 표시 ("O명 참여", 정원 숨김) |
| | 마감 임박/마감 상태 표시 |
| | 마이페이지 신청 내역 |

#### 검증 포인트

- [ ] 모임 신청 → 결제 완료까지 정상 동작
- [ ] 결제 완료 후 참가 인원 실시간 증가
- [ ] 취소 시 환불 규정에 따라 정확한 환불 금액 계산
- [ ] 부분 환불(50%) 정상 처리
- [ ] 정원 초과 시 대기 등록 가능
- [ ] 카카오 로그인 정상 동작

> **Note:** 계좌이체 결제는 운영자 확인이 필수이므로 M5에서 구현합니다.

#### 의존성 다이어그램

```
[M1 완료]
    ↓
[M2 완료] → M3, M4 동시 시작 가능
```

### M3: 알림 시스템

**목표:** 회원이 모임을 잊지 않게, 자동 리마인드 시스템 구축  
**📋 Work Package:** [WP-M3-알림시스템.md](./work-packages/WP-M3-알림시스템.md)

| 항목 | 내용 |
|------|------|
| **기간** | 1~2주 |
| **선행 조건** | M2 완료 |
| **핵심 가치** | 잊지 않게 |

#### 산출물

| 구분 | 산출물 |
|------|--------|
| **알림 인프라** | 솔라피 알림톡 연동 |
| | 알림 서비스 추상화 레이어 (추후 교체 용이) |
| | 알림 발송 기록 저장 (notification_logs) |
| **자동 알림** | 모임 리마인드 (3일 전, 1일 전, 당일) |
| | 대기자 자리 발생 알림 |
| | 응답 대기 시간 로직 (24시간/6시간/2시간) |
| | 월말 참여 독려 알림 |
| | 세그먼트별 리마인드 (온보딩 이탈 위험, 휴면 위험, 자격 만료 임박) |
| **스케줄링** | Vercel Cron 또는 Supabase pg_cron 설정 |
| | 대기자 응답 시간 초과 체크 |
| **운영자** | 수동 알림 발송 (공지용) |

#### 검증 포인트

- [ ] 모임 3일 전 리마인드 알림 발송됨
- [ ] 대기자에게 자리 발생 알림 발송됨
- [ ] 응답 대기 시간 초과 시 다음 대기자에게 알림
- [ ] 세그먼트별 리마인드 발송 (온보딩 이탈 위험, 휴면 위험, 자격 만료 임박)
- [ ] 운영자가 수동으로 알림 발송 가능
- [ ] 알림 발송 기록이 저장됨

#### 의존성 다이어그램

```
[M2 완료]
    ↓
[M3 완료] → M4와 병행 가능
    ↓
    → M5의 알림 템플릿 관리에 필요
```

### M4: 소속감 기능

**목표:** "나는 지독해 회원"이라는 느낌 강화  
**📋 Work Package:** [WP-M4-소속감기능.md](./work-packages/WP-M4-소속감기능.md)

| 항목 | 내용 |
|------|------|
| **기간** | 2~3주 |
| **선행 조건** | M2 완료 (M3과 병행 가능) |
| **핵심 가치** | 소속감 |
| **테스트** | Beta 테스트 시작 (활성 회원 20~30명) |

#### 산출물

| 구분 | 산출물 |
|------|--------|
| **참여 완료** | 모임 종료 3일 후 "어떠셨어요?" 알림 |
| | [칭찬하기] [참여했어요] [후기 남기기] 선택 화면 |
| | 7일 미응답 시 자동 참여 완료 처리 |
| | 노쇼 회원 "미참여" 처리 (운영자) |
| **칭찬하기** | 익명 칭찬 시스템 |
| | 선택형 문구 5개 |
| | 한 모임당 1명 칭찬 제한 |
| | 같은 사람 3개월 내 중복 칭찬 방지 |
| | 누적 칭찬 수 표시 |
| **건의하기** | 건의/피드백 제출 기능 |
| **후기 남기기** | 후기 작성 기능 |
| | "후킹페이지 공개 동의" 옵션 |
| **배지 시스템** | 첫 참여 배지 |
| | 10회 참여 배지 |
| | 연속 4주 참여 배지 |
| | 칭찬 배지 (10개/30개/50개) |
| **마이페이지 강화** | 총 참여 횟수 |
| | 연속 참여 기록 |
| | "함께한 지 OOO일째" |
| | 정기모임 자격 상태 ("마지막 참여: O월 O일") |
| | 6개월 임박 시 자격 만료 예정 알림 |
| | 배지 목록 |
| **참가자 목록** | 참가 인원 클릭 시 참가자 일부 공개 |
| | 칭찬 10개 이상 회원 우선 표시 |
| | 나머지 흐릿하게 처리 |
| **내 책장** | 책 등록 (제목, 저자) |
| | 한 문장 기록 (선택) |
| | 같은 책 중복 등록 방지 |
| | 총 등록 책 수 표시 |

#### 검증 포인트

- [ ] 모임 종료 3일 후 알림 발송 → 선택 화면 표시
- [ ] 칭찬하기 → 누적 칭찬 수 증가
- [ ] 참여 완료 시 배지 조건 체크 → 배지 부여
- [ ] 마이페이지에서 참여 통계 확인 가능
- [ ] 참가자 목록에서 칭찬 많이 받은 회원 우선 표시
- [ ] 책장에 책 등록 및 한 문장 기록 가능

#### 의존성 다이어그램

```
[M2 완료]
    ↓
[M4 완료] → M5와 함께 진행 가능
    ↓
    → M6의 후킹페이지 후기 표시에 필요
```

### M5: 운영자 도구 + 계좌이체

**목표:** 운영자 부담 최소화, 효율적인 운영 지원, 계좌이체 결제 옵션 제공

|------|------|
| **기간** | 2~3주 |
| **선행 조건** | M3 완료 (알림 템플릿용), M4 완료 권장 |
| **핵심 가치** | 신뢰 (운영 관점) |

#### 산출물

| 구분 | 산출물 |
|------|--------|
| **대시보드** | 이번 달 참가 현황 |
| | 이번 달 수입 |
| | 환불 내역 |
| | 재참여율 통계 |
| | 세그먼트별 회원 수 |
| | 입금대기/환불대기 건수 표시 |

| | 계좌 정보 표시 및 클립보드 복사 |
| | 입금자명 형식 "MMDD_이름" 안내 |

| **계좌이체 (운영자)** | 입금대기 목록 (모임별 필터, 입금자명 검색) |
| | 입금 확인 버튼 → 참가 확정 처리 |
| | 입금자명 불일치 시 수동 매칭 |
| | 환불대기 목록 + 환불 완료 체크 |

| | 발송 시점 조정 |
| | 특정 알림 on/off |
| **권한 관리** | 다른 운영자에게 권한 선택적 부여 |
| | 권한 종류: 모임 관리, 알림 발송, 배너 관리, 요청함 답변, 대시보드 열람, 템플릿 관리 |
| **요청함** | 문의 확인 |
| | 답변 기능 |
| **배너 관리** | 홈 화면 배너 등록/수정/삭제 |
| | 배너 순서 관리 |

- [ ] 대시보드에서 이번 달 참가/수입/환불 확인 가능
- [ ] 알림 문구 수정 후 변경된 문구로 발송됨
- [ ] 다른 운영자에게 특정 권한만 부여 가능
- [ ] 문의에 답변 가능
- [ ] 홈 화면 배너 변경 시 즉시 반영
- [ ] 계좌이체 선택 시 계좌 정보 표시 및 클립보드 복사 동작
- [ ] "입금했습니다" 체크 후 입금대기 상태로 전환
- [ ] 입금대기 시 정원 즉시 예약됨 (다른 사용자 차단)
- [ ] 24시간 미입금 시 자동 취소 및 정원 복구
- [ ] 운영자 입금 확인 시 참가 확정 전환
- [ ] 계좌이체 취소 시 환불 계좌 수집 모달 동작
